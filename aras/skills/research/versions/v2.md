# Research Planner Skill

## Core Task
Given a research topic, produce a **STRICT JSON** plan with:

- **hypothesis** — 1 sentence, falsifiable claim with specific technical mechanism (e.g., "via gradient covariance estimation")
- **questions** — 3–5 specific, falsifiable research questions answerable by the experiments
- **experiments** — 3 experiment names including at least one ablation study tied to the hypothesis mechanism
- **metrics** — list of measurable outcomes with units (e.g., "wall_time_seconds", "expected_calibration_error")
- **outline** — list of paper section titles (standard academic structure)
- **section_briefs** — **REQUIRED** object mapping each outline title to a 2-3 sentence description of specific technical content to be included (equations, algorithms, key results, architectures). Must prevent empty sections.
- **keywords** — 8–12 search terms including both general and specific technical terms
- **domain** — one of: `nlp`, `computer_vision`, `reinforcement_learning`, `general_ml`, `theory`, `systems`, `bioinformatics`, `other`

## Output Format
Return **JSON only**. No markdown. No preamble. No explanation.

```json
{
  "hypothesis": "We hypothesize that ...",
  "questions": ["Q1", "Q2", "Q3"],
  "experiments": ["exp1_baseline", "exp2_ablation_mechanism", "exp3_robustness"],
  "metrics": ["accuracy", "f1", "runtime_seconds"],
  "outline": ["Abstract", "Introduction", "Related Work", "Methodology", "Experiments", "Results", "Discussion", "Conclusion"],
  "section_briefs": {
    "Abstract": "150-200 words summarizing the uncertainty quantification problem, proposed ensemble/Bayesian approach, key empirical results (ECE reduction, OOD detection AUROC), and significance.",
    "Introduction": "3 paragraphs: (1) Motivation for uncertainty quantification in safety-critical deep learning, (2) Specific gap: existing ensembles lack calibration vs. Bayesian methods, (3) 4 contributions: deep ensemble analysis, computational trade-off analysis, OOD detection benchmark, calibration metric comparison.",
    "Methodology": "Mathematical formulation: deep ensemble variance estimation Var(x) = 1/M sum(f_m(x) - f_bar(x))^2, Bayesian approximation via variational inference with ELBO loss, network architectures (ResNet-18, Transformer), training hyperparameters.",
    "Experiments": "Implementation details of 3 experiments with dataset specifications (CIFAR-10, ImageNet-C), hardware specs, and statistical testing methodology.",
    "Results": "Tables showing ECE, NLL, Brier score comparisons; figures for reliability diagrams; ablation analysis of ensemble size M.",
    "Discussion": "Interpretation of why ensembles outperform VI in high-uncertainty regions, limitations regarding computational cost, future directions for scalable Bayesian methods.",
    "Conclusion": "Summary of findings regarding ensemble superiority for calibration, practical recommendations for ensemble size selection."
  },
  "keywords": ["keyword1", "keyword2", "...", "keyword10"],
  "domain": "general_ml"
}
```

## Quality Criteria
- Hypothesis **must be falsifiable** — contains "we hypothesize", "we propose", or "we demonstrate" AND specifies the technical mechanism (e.g., "via diagonal Hessian approximation", "using deep ensembles of size M≥5")
- **Section briefs are mandatory** and must contain specific technical markers: mathematical notation (e.g., "L(θ)", "∇"), algorithm names (e.g., "SGD with momentum", "Variational Inference"), or concrete architectural details (e.g., "ResNet-18 backbone")
- Abstract brief must specify word count (150-200) and mention contributions, methods, and key findings
- Introduction brief must explicitly mention "motivation", "gap", and list 3-4 specific contributions
- Methodology brief must include at least one equation description or algorithmic procedure
- Experiments must include at least one ablation study targeting the hypothesis mechanism (e.g., "ablation_ensemble_size", "ablation_curvature_proxy")
- Keywords must include both **general terms** AND **specific technical terms** (e.g., both "machine learning" and "gradient boosting calibration")
- Domain must match the most prominent keywords
- Output must be valid parseable JSON — no trailing commas, no comments, no markdown code blocks
